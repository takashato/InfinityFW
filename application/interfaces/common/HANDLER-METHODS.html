Handler Methods

Methods are children of the handler.

How do they work?

Let's look the current URL, we're at ./{$smarty.const.REQSTR}.
That means we invoked the handler Handlers, and its method Methods. Other parts are attributes.
Notice: If no method is indicated, or method is not available, the method index will be invoked.
If method index is not available, an error will appear.

What are they based on?

They are also methods of handler class.
A handler can contain a lot of methods, but at least have ONE method (index method).
They must be public method. 
namespace Handler;
class Example extends Handler{
    public function index($attr1, $attr2, $attr3,...){
        // Index method
    }

    public function Methods($attr1, $attr2, $attr3,...){
        // Method handler
    }
}

Wait...!! What are attributes?

Let's look this ./Handlers/Methods/attr1/attr2/attr3/....
attr1, attr2, attr3,... are attribute values. They will be used as arguments of method. And they are unlimited.
Suppose we are at ./Handlers/Methods/Takashato/Vietnam/Male, and our method is below. 

public function Methods($attr1, $attr2, $attr3){
    echo 'Name: '.$attr1.'. ';
    echo 'Nationality: '.$attr2.'. ';
    echo 'Gender: '.$attr3.'.';
}
Here we go, we will get the output Name: Takashato. Nationality: Vietnam. Gender: Male.
